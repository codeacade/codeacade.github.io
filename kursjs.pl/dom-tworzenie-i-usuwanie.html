Tworzenie i usuwanie elementów

Potrafimy już poruszać się po elementach, potrafimy je pobrać, przechodzić z jednego na drugi itp. Czas zacząć je tworzyć a potem usuwać. Nie jest to wcale takie trudne. Wystarczy, że skorzystamy z kilku podstawowych metod oferowanych przez JavaScript.

Zanim przejdziemy dalej, mikro powtórka. HTML składa się z węzłów, czyli powiedzmy kawałków. Część z nich jest elementami html (tagami - np. span, strong), inna część najczęściej jest tekstem (np. lorem ipsum).

Aby stworzyć pojedynczy element (np. span) skorzystamy z funkcji createElement. Do tworzenia węzłów tekstowych możemy użyć między innymi funkcji createTextNode (ale też i właściwości np. innerText).

Tworzenie obiektu za pomocą createElement

Aby utworzyć nowy element na stronie korzystamy z metody document.createElement(typ). Po utworzeniu nowego elementu możemy na nim działać tak jakbyśmy go wcześniej pobrali za pomocą querySelector, czyli obowiązują nas te same zasady co wcześniej. Taki element jest totalnie pusty, dlatego przed wstawieniem dodajmy mu kilka rzeczy: 


const el = document.createElement("div");

el.id = "myDiv";
el.innerText = "Tekst w divie";
el.setAttribute("title", "To jest tekst w dymku");
el.classList.add("module");
el.style.setProperty("background-color", "#FF6633");

Wstawianie elementu za pomocą appendChild

Ale to nie wszystko. Utworzony w powyższy sposób element jest dostępny dla skryptu, ale nie ma go jeszcze w drzewie dokumentu. Musimy go tam wstawić. Jedną z możliwości jest użycie metody: parentElement.appendChild(nowyElement): 


const el = document.createElement("div");

el.id = "myDiv";
el.innerText = "Tekst w divie";
el.setAttribute("title", "To jest tekst w dymku");
el.classList.add("module");
el.style.backgroundColor = "#FF6633";

const div = document.querySelector(".test-first"); //pobieramy miejsce docelowe
div.appendChild(el); //wstawiamy element do drzewa dokumentu

No dobrze. Ale po co takie dzikie kombinowanie, skoro mogę po prostu użyć innerHTML i wszystko wpisać z palca, tak jakbym pisał html? 


const elHtml = '<div id="bloczek" title="To jest tekst w dymku" class="uber-element" style="background-color:#FF6633">Tekst w divie</div>';

const div = document.querySelector("parent");
div.innerHTML = elHtml

Faktycznie za pomocą innerHTML wstawiasz kod html w dany element. Ale czy właściwość ta daje ci referencję do elementów we wstawianym html? Bardzo często będziesz chciał za chwilę zrobić jakąś akcję na wstawianych elementach - np. podpiąć im kliknięcie, zmienić im tekst itp. Jeżeli będziesz korzystał wyłącznie z innerHTML, będziesz musiał te elementy po wstawieniu dodatkowo pobrać - np. za pomocą querySelector. Nie zawsze jest to wygodniejsze, a czasami wręcz niemożliwe.

Wstawianie elementu za pomocą insertBefore

Do wstawiania nodów w drzewo strony możemy też skorzystać z metody parentNode.insertBefore(newElement, element), która wstawia nowy element przed wskazanym: 


<p id="beforeTest">
    <strong>Jestem pierwszym elementem</strong>
</p>


let counter = 0;
const btn = document.querySelector("button");

btn.addEventListener("click", function() {
    const p = document.querySelector("#beforeTestP");
    const strong = p.firstElementChild;

    const newEl = document.createElement("span"); //tworzymy nowy nod z tekstem
    newEl.innerText = `Inny element ${++counter}`;

    p.insertBefore(newEl, strong); //wstawiamy go przed pierwszym elementem
});

Jestem pierwszym elementem

Tworzenie tekstu za pomocą createTextNode

Wspomnieliśmy powyżej, że html składa się głównie z 2 bytów: elementów html i tekstów (ale też np. atrybutów).

Aby dodać tekst na stronę możemy tak jak powyżej używać właściwości innerText/textContent, innerHTML, ale też możemy utworzyć pojedynczy węzeł tekstowy za pomocą funkcji createTextNode


const text = document.createTextNode("Lubię placki");
element.appendChild(text);

//lub
element.innerText = "Lubię placki";

Ale po co takie kombinowanie, skoro użycie innerText jest zwyczajnie wygodniejsze? Już biegnę z wyjaśnieniami.

Spójrzmy na poniższy przykład:


<p>
Psy <strong>są</strong> fajne
</p>

Strong jest elementem. Słowo "Psy" jest węzłem tekstowym, tak samo jak słowo "fajne". Podobnie zresztą węzłem tekstowym jest słowo "są", które znajduje się w strong.

W powyższych przykładach gdy tworzyliśmy elementy, ich tekst ustawialiśmy za pomocą właściwości innerHTML lub innerText. Takie podejście jest super wygodne, ale czasami może nie wystarczyć. Co gdybyśmy w powyższym przykładzie chcieli dynamicznie zmieniać słowa "Psy" i "fajne"? Rozwiązań jest oczywiście kilka. Możemy je wygenerować jako spany i używać na nich innerText.


<p>
<span>Psy</span> <strong>są</strong> <span>fajne</span>
</p>

A co jeżeli nie moglibyśmy dodać żadnego nowych elementów, bo zaburzyły by stylowanie?
Możemy też utworzyć te węzły za pomocą createTextNode(), a dzięki referencji (bo tworzymy je jako zmienne) bez problemu będziemy zmieniać ich zawartość:


const p = document.querySelector("p");
const btn = document.querySelector('.button');

const word1 = document.createTextNode("Psy");
p.appendChild(word1);

const strong = document.createElement("strong");
strong.innerText = " są "; //tu nie potrzebujemy referencji pod zmienną
p.appendChild(strong);

const word2 = document.createTextNode("fajne");
p.appendChild(word2);

btn.addEventListener("click", function() {
    console.dir(word1); //wypiszemy sobie by zobaczyć co możemy użyć

    word1.nodeValue = "Koty też";
    word2.nodeValue = "super!";
});

Zbadaj poniższy paragraf, a następnie kliknij w przycisk:

append, prepend, before i after

jQuery - jedna z najlepszych rzeczy jaką wymyślił człowiek udoskonaliła wiele mechanizmów jakie występują w czystym JavaScript - chociażby dopiero co omawiany przez nas dataset (1, 2), ale i wiele innych takich jak parents, closest itp.

Podobnie jeżeli chodzi o wstawianie elementów do dokumentu. Od dawien dawna występowały tam metody append(), prepend(), before() i after()., których używa się zwyczajnie o wiele wygodniej niż jakiś appendChild czy insertBeforów.

W ostatnim czasie JavaScript kilka z takich pomysłów wchłonął i do siebie. Mamy więc typowy odpowiednik remove(), closest() czy właśnie omawiane metody, które możemy użyć do wstawiania elementów ale i całych tekstów na stronę:

append() wstawia treść/element na koniec danego elementu
prepend() wstawia treść/element na początku danego elementu
before() wstawia treść/element przed danym elementem
after() wstawia treść/element za danym elementem 

const cnt = document.querySelector(".container");
let counter = 0;

const btnBefore = document.querySelector("#test-before");
btnBefore.addEventListener("click", function() {
    const strong = document.createElement("strong");
    strong.innerHTML = "Przed " + ++counter;
    strong.classList.add('element');
    cnt.before(strong);
});

const btnAfter = document.querySelector("#test-after");
btnAfter.addEventListener("click", function() {
    const strong = document.createElement("strong");
    strong.innerHTML = "Za " + ++counter;
    strong.classList.add('element');
    cnt.after(strong);
});

const btnPrepend = document.querySelector("#test-prepend");
btnPrepend.addEventListener("click", function() {
    const strong = document.createElement("strong");
    strong.innerHTML = "Na początku " + ++counter;
    strong.classList.add('element');
    cnt.prepend(strong);
});

const btnAppend = document.querySelector("#test-append");
btnAppend.addEventListener("click", function() {
    const strong = document.createElement("strong");
    strong.innerHTML = "Na końcu " + ++counter;
    strong.classList.add('element');
    cnt.append(strong);
});

W przeciwieństwie do insertBefore i appendChild powyższe metody możemy też użyć do wstawiania tekstów. Przekazany w ten sposób html będzie wstaiony także jako tekst:


    element.append("Ten span - o ten: <span></span> będzie zapisany jak tekst");

Ostatnia mini różnica rozróżniająca appendChild i insertBefore od nowszych braci to to, że te pierwsze zwracają w rezultacie element, co daje nam możliwość używania method chaining. Te nowsze (append, prepend, before, after) zwracają undefined, co powoduje, że pisanie "na skróty" nie zadziała: 


const newEl = document.createElement("div");
document.body.appendChild(newEl).classList.add("element");

const newEl = document.createElement("div");
document.body.append(newEl).classList.add("element"); //błąd - Cannot read property 'classList' of undefined

Osobiście nie zawracał bym sobie tym głowy, bo pamięć nasza jest ograniczona, a i takie chodzenie na skróty często nie kończy się dobrze. Zamiast tego piszmy ładny - może i ciut dłuższy - ale i czytelniejszy kod: 


const newEl = document.createElement("div");
document.body.appendChild(newEl);
newEl.classList.add("element");

Klonowanie elementów

Za pomocą funkcji cloneNode(deep) możemy wykonać kopię danego elementu.


<div class="to-clone">
    <strong>Testowy</strong>
</div>


const el = document.querySelector(".to-clone");

const cloneEl1 = el.cloneNode();
console.log(cloneEl1); //<div class="to-clone"></div>

const cloneEl2 = el.cloneNode(true);
console.log(cloneEl2); //<div class="to-clone"><strong>Testowy</strong></div>

Po sklonowaniu elementu możemy na nim działa tak samo jakbyśmy go stworzyli od nowa, albo pobrani za pomocą dowolnej metody.


const btn = document.querySelector("button");
const targetDiv = document.querySelector(".target");
let nr = 0;

btn.addEventListener("click", function() {
    nr++;
    const el = document.querySelector('.to-clone');
    const clone = el.cloneNode(true);
    clone.style.background = "dodgerblue";
    clone.querySelector('.element-name').innerText = `Element ${nr}`
    targetDiv.appendChild(clone);
})
Element

Niestety klonowany za pomocą cloneNode() jest tylko kod html elementu - bez podpiętych pod niego eventów. Powoduje to, że nie da się tej metody swobodnie stosować na dowolnym elemencie, bo potencjalnie stracimy podpiętą pod niego logikę.

Niestety nie ma łatwego sposobu na obejście tego. Albo napiszemy dodatkową logikę, albo podepniemy eventy raz jeszcze, albo... użyjemy jQuerowego odpowiednika clone(), który pozwala klonować elementy wraz z eventami.

Usuwanie elementów

Element, który istnieje w drzewie dom, możemy z niego usunąć na kilka sposobów. Możemy skorzystać z element.remove(), która usuwa element lub parentNode.removeChild(element) 


<div class="div-test-remove">
    <span>Element do usunięcia</span>
</div>


const div = document.querySelector("div")
const element = div.querySelector("span");

div.removeChild(element);

//lub
element.parentElement.removeChild(element);

//lub
element.remove();
Element do usunięcia

Aby usunąć wszystkie dzieci danego elementu - czyli go wyczyścić, powinniśmy wykonać pętlę po jego dzieciach i wszystkie pousuwać: 


const ul = document.querySelector("#list");

while (ul.firstChild) {
    ul.removeChild(ul.firstChild); //lub div.firstChild.remove()
}

Jeżeli chcemy usuwać elementy za pomocą pętli for, wtedy róbmy ją od końca, ponieważ po usunięciu danego elementu, może nam się przestawić indeks w kolekcji.


const li = document.querySelectorAll("#list li");
for (let i=li.length-1; i<=0; i++) {
    li[i].remove();
}

...można też nieco prościej:


const ul = document.querySelector("#list");
ul.innerHTML = "";

Usuwanie z html i wycieki pamięci

Metody removeChild() i remove() usuwają elementy z HTML, ale jeżeli taki element jest podstawiony pod zmienną, po usunięciu dalej będzie ona przetrzymywać dany element w pamięci.

Przy pojedynczych usuwanych elementach nie powinno to być raczej problemem.

Problem pojawi się jednak, gdy takich elementów będziemy pobierać/tworzyć i usuwać bardzo wiele (np. w setInterval).

Element 1 Element 2 Element 3 Element 4

Otwórz konsolę by kliknąć i usunąć powyższe elementy 


const btn = document.querySelector("button");
const elToDelete = document.querySelectorAll('.el-to-delete');

btn.addEventListener("click", function() {
    for (const el of elToDelete) {
        el.remove(); //usuwamy tylko z HTML
    }

    //elementy wciąż są w pamięci pod zmienną elToDelete
    console.log(elToDelete);
})

Sprawdź też omawiany problem na przykładowej stronie: delete-element-memory-test.html

Rozwiązaniem jest pozbywać się takich referencji. Możemy dane zmienne ustawić na null, czyścić tablice w których się znajdują (np. poprzez ustawienie ich na pustą tablicę [] lub ich length na 0) itp.

Zastępowanie elementów

Do zastępowania jednego elementu drugim użyjemy metody parent.replaceChild(newChild, oldChild): 


<div class="div-test-replace">
    <span>Jestem starym elementem</span>
</div>
<button type="button" class="button" id="replaceTest">Zastąp spana nowym elementem</button>


const div = document.querySelector(".div-test-replace")
const btn = document.querySelector("#replaceTest");

btn.addEventListener("click", function() {
    const oldItem = div.querySelector("span");

    const newItem = document.createElement("strong");
    newItem.innerText = "Jestem nowym elementem";

    div.replaceChild(newItem, oldItem);
});
Jestem starym elementem

Tworzenie fragmentów dokumentu

Korzystając z powyższych metod z łatwością możemy tworzyć różne skomplikowane kawałki drzewa. Możemy do tego celu wykorzystać także metodę createDocumentFragment(), która tworzy fragment dokumentu. Fragment taki jest "zbiornikiem", podobnym do zwykłego diva, jednak przy wstawianiu go do dokumentu, wstawiane zostają tylko jego dzieci.

Ale po co to komu? Komu to potrzebne? Wydajności. Gdy wstawiasz do html nowy element, drzewo DOM musi zostać przebudowane. Gdy wstawiasz takich elementów wiele, za każdym razem to drzewo będzie przebudowywane. Jeżeli w danej chwili zależy ci bardzo na wydajności (bo na przykład robisz szybką grę), powinieneś raczej wstawiać dane elementy "paczkami". 


const fragment = document.createDocumentFragment();

for (let i=0; i<10; i++ ) {
    const p = document.createElement("p");
    p.appendChild(document.createTextNode("Akapit "+(i+1)));
    fragment.appendChild(p);
}

document.body.appendChild(fragment); //wstawiamy 10 paragrafów

I teraz pytanie do ciebie?
Czy znasz inną technikę wstawiania wielu elementów na raz?
A innerHTML znasz? Spokojnie możesz wykorzystać tą właściwość by do elementu wstawiać skomplikowany kod.
Pamiętaj tylko o tym, że używając tej metody wstawiasz html do strony, ale metoda ta nie utworzy ci automatycznie referencji...

Trening czyni mistrza

Poniżej zamieszczam kilka zadań, które w ramach ćwiczenia możesz wykonać:

  1. Pobierz sobie stronę stąd (prawy klawisz na linku, zapisz jako).

    Skrypty pisz inline w kodzie strony - na końcu body.

  2. Pobierz element .list i podstaw go pod zmienną.
    Za pomocą pętli stwórz 5 elementów div i wstaw je do pobranej listy.

    Każdy stworzony div powinien mieć:
    - klasę .element
    - atrybut title ustawiony na "jestem divem numer ..."
    - tekst w środku "Element ..."

    W miejsce kropek wpisz kolejne numery divów 

    
                    const list = document.querySelector(".list");
                    for (let i=0; i<5; i++) {
                        const div = document.createElement("div");
                        const nr = i+1;
                        div.classList.add("element");
                        div.setAttribute("title", "Jestem divem numer " + nr);
                        div.innerText = "Element " + nr;
    
                        list.appendChild(div);
                    }

  3. W kodzie masz przygotowane zdarzenie click dla buttona .delete.
    Napisz w nim treść skryptu, która usunie elementy dodane w 2 zadaniu.
    Jeżeli nie ma elementów w liście, niech wyskoczy w konsoli tekst "Nie mam co usunąć" 

    
                document.querySelector("#delete").addEventListener("click", function() {
                    //tutaj napisz usuwanie elementów w liście
                    const list = document.querySelector(".list");
                    const div =  list.querySelectorAll("div");
                    for (let i=div.length-1; i>=0; i--) {
                        div.remove();
                        //lub
                        //list.removeChild(div);
                    }
                })
